디자인 (UI&UX)

코드 개발

왜 프로젝트를 시작하게 되었는가?

우리 팀의 프로젝트는 웹 개발 실무 능력을 키우기 위해 기획되었습니다. 단순한 디자인 복제에서 벗어나 팀원들의 아이디어를 담은 개성 있는 페이지를 만들고자 했습니다. 사용자 경험을 고려한 UI/UX 설계와 팀 협업을 통해 실제 서비스처럼 완성도를 높이는 것을 목표로 했습니다.

이훈진

멋쟁이사자처럼 프론트엔드 13기

다른 페이지의 디자인을 그대로 구현하는 클론 코딩보다는 좀 더 팀원들의 개성을 보여줄 수 있는 주제가 없을까 하면서 다같이 의논했습니다. 그 결과 간단하게 우리 팀을 소개하는 페이지를 만들면 어떨까 하는 의견이 나와 팀 소개 페이지를 만들게 되었습니다. 화려한 페이지는 아니지만 심플한 디자인 컨셉에 충실한 페이지가 잘 구현된 것 같아 뿌듯합니다!

강석현

멋쟁이사자처럼 프론트엔드 13기

프로젝트 기획은 어떻게 했을까?

최근 프론트엔드 개발자분들의 포트폴리오를 찾아보며 공부하는 과정에서, 많은 개발자들이 자신의 포트폴리오를 단순한 PT 자료가 아닌 웹사이트나 블로그 서비스 형태로 제작해 본인의 경험을 보여주는 사례를 많이 접했습니다. 이러한 트렌드를 보며 저희도 앞으로의 성장을 기록하고, 자신만의 포트폴리오를 구축할 필요성을 느꼈습니다.
비록 지금은 부족한 부분이 있지만, 앞으로 저희가 만든 웹사이트를 돌아보며 "아, 이만큼 성장했구나"라고 느낄 수 있는 계기가 되었으면 하는 바램이 있습니다. 또한, 기존의 시안을 활용하는 것이 아닌 기획부터 UI 디자인까지 직접 설계함으로써, 앞으로 진행할 다양한 프로젝트에 대비하는 경험을 쌓기 위해 이번 프로젝트 주제를 선정하게 되었습니다.
저희팀은 설계적 측면과 사용자 경험을 동시에 고려하며 디자인을 진행했습니다. 컴포넌트를 구축해 코드의 재사용성을 높였으며, 다양한 스타일을 적용해 동일한 컴포넌트라도 각기 다른 느낌을 주도록 의도했습니다. 또한, 모바일부터 웹까지 전반적인 반응형 디자인을 적용해 사용자가 어떤 화면에서도 서비스를 편리하게 이용할 수 있도록 설계했습니다. 시각적 측면에서는 파란색 계열을 포인트 컬러로 사용해 웹 페이지에 처음 방문했을 때 시원하고 정돈된 느낌을 전달하도록 디자인했습니다. 파란색은 강렬한 인상을 주면서도 과도하게 자극적이지 않아, 사용자에게 편안하면서도 집중을 유도하는 효과를 기대할 수 있도록 의도했습니다.
저희팀은 UI 프로젝트라는 주제에 맞게 JavaScript 없이, 그동안 수업시간에 배운 HTML과 CSS를 적극 활용하였습니다. 특히, 일반 CSS가 아닌 Tailwind CSS를 사용하여 아직 익숙하지 않은 기술에 도전하며 실무에서의 활용 능력을 키우고자 했습니다. 또한, 개발 과정에서는 Git과 GitHub를 통해 협업을 진행했습니다. 무작정 개발을 진행하는 것이 아니라, 매일 첫 시간에 각자 맡은 파트의 진행 상황, 이슈, 금일 진행 예정 작업 등을 공유하는 아젠다를 설정해 보다 체계적으로 개발을 진행할 수 있었습니다. 개발 도중 이슈가 발생했을 경우, 모든 팀원이 상황을 즉각 공유하고 함께 해결하는 방식을 통해 문제를 효과적으로 해결하며 원활한 개발 환경을 유지했습니다.

프로젝트 마무리 소감

안녕하세요 이훈진입니다.

이번 프로젝트를 통해 단순히 주어지는 시안이 아닌 팀원 모두가 참여하여 기획부터 디자인, 개발, 배포까지 모든 과정을 경험하여 굉장히 뜻깊은 프로젝트였던 것 같습니다. 이전에 부족했던 협업 툴 사용과 기록하는 습관을 기르게 되었습니다. 아직 부족하지만 더욱 성장하여 목표를 위해 달려가겠습니다!!

안녕하세요 배희정입니다.

멋사 13기에서 첫 프로젝트를 진행하며 프로젝트를 시작하기 위해 필요한 준비, 여러가지 상황에서 git을 사용하는 방법, 프로젝트 문서 정리를 위해 github를 활용하는 방식 등 많은 것들을 배우게 되었습니다. 이번 경험을 토대로 다음 프로젝트에서는 더 발전된 모습을 보여드리고 싶습니다!

안녕하세요 배동초입니다.

이번 프로젝트를 통해 협업과 소통, 그리고 배려의 중요성을 배우며, 기획 등 스스로 만들어가는 프로젝트의 의미를 이해했고, 앞으로도 기획력과 Git 관련 활용 능력을 더욱 발전시킬 것을 다짐합니다. 함께해 준 팀원들에게 진심으로 감사드립니다.

안녕하세요 강석현입니다.

안녕하세요, 강석현입니다. 이번 프로젝트를 통해 Git과 GitHub을 활용하며 협업과 소통의 중요성을 깊이 배웠고, Git 충돌 해결 과정을 통해 동작 원리를 이해하게 되었습니다. 앞으로도 꾸준히 학습하며 이러한 도구를 더 효과적으로 사용할 수 있도록 노력하겠습니다.
